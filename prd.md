# Product Requirements Document (PRD): 個人投資追蹤系統 (Personal Investment Tracker)

## 1. 產品概述 (Product Overview)
本產品為一個供個人使用的網頁應用程式 (Web App)，旨在幫助使用者輕鬆追蹤與管理個人投資組合。系統將整合各項資產部位，特別關注於股息再投入的流程管理，並提供直觀的資產配置圖表，讓使用者能隨時掌握財務狀況並做出適當的投資決策。

## 2. 目標受眾 (Target Audience)
*   **主要使用者**：產品擁有者本人。
*   **需求痛點**：
    *   持有多種資產（台股、美股、債券等），難以快速看出整體資產配置比例。
    *   領取股息後，容易忘記哪些資金需要再投入，或者忘記已經將哪些股息投入了特定標的，導致資金閒置或追蹤混亂。

## 3. 核心功能需求 (Core Features)

### 3.1 投資部位管理 (Holdings Management)
使用者可以透過 UI 檢視、新增、編輯及刪除各項投資標的與現金部位。
*   **資產類別 (Asset Classes)**：系統必須支援以下分類：
    *   台股 (Taiwan Stocks)
    *   美股 (US Stocks)
    *   債券 (Bonds)
    *   外幣 (Foreign Currency)
    *   現金 (Cash)
*   **紀錄欄位**：
    *   標的名稱 / 代號 (例如：AAPL, 0050)
    *   資產類別 (由上述類別中選擇)
    *   持有數量 (股數 / 金額)
    *   (進階) 平均成本、目前市價
*   **UI 互動**：
    *   提供總覽列表。
    *   提供新增按鈕與表單。
    *   提供調整數量的介面 (例如：買入/賣出增加減少數量)。

### 3.2 股息與再投入管理 (Dividend & Reinvestment Tracking)
追蹤股息收入，並管理其「再投入」的狀態，確保資金充分利用。
*   **股息紀錄輸入**：
    *   當收到股息時，可透過 UI 輸入：發放日期、發放標的、發放金額。
    *   新增紀錄時，預設狀態為「待再投入」(Pending Reinvestment)。
*   **待投入資金池 (Reinvestment Pool)**：
    *   專屬區塊顯示目前有哪些股息處於「待再投入」狀態，並加總未投入的總金額。
*   **再投入狀態調整**：
    *   使用者可透過 UI 將特定的「待再投入」股息紀錄，標記為「已再投入」(Reinvested)。(考慮到下單系統，再投入的金額不一定是全額)
    *   標記為「已再投入」時，系統需提供欄位讓使用者輸入或選擇「再投入標的」(Reinvested Target) 及「再投入日期」。
    *   (加分項) 當標記為再投入時，系統詢問是否連動更新「3.1 投資部位管理」中的數量。

### 3.3 資產配置視覺化 (Asset Allocation Dashboard)
提供視覺化圖表，讓使用者一目了然目前的資產比例，以利進行資產再平衡。
*   **圖表呈現**：圓餅圖 (Pie Chart) 或甜甜圈圖 (Doughnut Chart)。
*   **分類維度**：精準依據「台股 / 美股 / 債券 / 外幣 / 現金」顯示各類別的佔比 (%)。
*   **價值計算**：
    *   資產佔比需基於「目前價值」。(目前價值 = 持有數量 x 目前市價 / 匯率)。
    *   串接外部 API (如 Yahoo Finance) 自動取得市價與匯率，自動計算總值。

## 4. 非功能需求 (Non-Functional Requirements)
*   **使用者介面 (UI/UX)**：簡潔、直觀且具現代感的設計 (例如使用 Material Design 或 Tailwind CSS)。需支援響應式設計 (RWD)，確保在手機與電腦上皆有良好的操作體驗。
*   **資料儲存 (Data Storage)**：將資料直接儲存於server。
*   **技術棧建議 (Tech Stack)**：
    *   前端框架：React, Vue 或 Svelte (建議依使用者熟悉度決定)。
    *   樣式：Tailwind CSS。
    *   圖表：Chart.js, Recharts 或 ECharts。

## 5. 階段性開發計畫 (Phased Development Plan)

### Phase 1: 核心功能與單機版 (MVP)
*   建立前端專案。
*   完成「投資部位」的 CRUD 功能 (儲存於 LocalStorage)。
*   完成「股息紀錄」的輸入與狀態切換功能 (待投入 / 已投入某標的)。
*   完成圓餅圖：根據手動輸入的市值或簡單加總計算資產比例。

### Phase 2: 自動化與連動
*   實作市價與匯率的簡單自動更新 (例如串接免費 API)，讓圓餅圖的佔比更精確即時。
*   實作股息「再投入」時自動增加目標標的「持有數量」的連動機制。

### Phase 3: 進階分析 (Future Scope)
*   歷史資產總值變化折線圖。
*   年度股息預估與歷年股息成長圖表。
*   雲端資料備份或匯出/匯入功能。
